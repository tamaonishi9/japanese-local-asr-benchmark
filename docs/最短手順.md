# 最短手順

録音した同じ音声を `small` / `large-v3` / `Kotoba-Whisper v2.0-faster` で比較するまでの最短手順です。Windows PowerShell で確認しています。

## CPU で試す

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
Copy-Item .\config.toml.cpu .\config.toml -Force
python record_and_compare.py
```

操作:

1. `Enter` で録音開始
2. 話し終えたら `Enter` で停止
3. `results/` の Markdown / CSV / JSON を確認

初回実行時はモデルをダウンロードするため、時間がかかります。

## Windows でモデル取得が止まる場合

次のエラーが表示された場合:

```text
[WinError 1314] クライアントは要求された特権を保有していません。
```

symlink を使わず再実行します。

```powershell
$env:HF_HUB_DISABLE_SYMLINKS = "1"
python record_and_compare.py
```

この設定ではモデルキャッシュのディスク使用量が増える場合があります。

## CUDA で試す

追加要件:

- NVIDIA GPU
- CUDA Toolkit 12.x
- cuDNN 9 for CUDA 12

CUDA Toolkit 12.x をインストールした後、仮想環境へ cuDNN を追加します。

```powershell
.\.venv\Scripts\Activate.ps1
python -m pip install nvidia-cudnn-cu12
$cudnnBin = (Resolve-Path .\.venv\Lib\site-packages\nvidia\cudnn\bin).Path
$env:PATH = "C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v12.x\bin;$cudnnBin;$env:PATH"
where.exe cublas64_12.dll
where.exe cudnn*.dll
Copy-Item .\config.toml.cuda .\config.toml -Force
python record_and_compare.py
```

`v12.x` はインストールした CUDA Toolkit の実際のバージョンへ置き換えてください。CUDA profile は計測前に warm-up を行うため、初回 GPU 初期化時間を RTF に含めず比較します。

詳細な設定・出力形式・モデル追加方法は [README](../README.md) を参照してください。
